##我们做什么？
我们从事互联网和智能电视交叉领域，解决大屏幕电视机方便、快捷访问互联网内容。

如果你听说过 Apple TV 或者 Google TV，那么我们跟他们所属一个领域。不过他们把智能系统整合在电视中，而我们把智能系统整合在一个盒子（或称为迷你电脑）里。

用户可舒服地坐在客厅沙发上通过摇控器，操作盒子里运行的 App 应用访问互联网内容，比如欣赏网络高清视频、浏览图片资讯、收听音乐等等。这些 App 应用所展现的丰富多彩内容，是由我们先在互联网垂直整合用户感兴趣的内容，通过网络直接推送到用户面前。

可能你会觉得这个和目前 IPTV 机顶盒、数字电视机顶盒没什么太大差别。但是我们在用户体验等很多方面有很大改进和创新。

很抱歉，目前我不能够透露的太多的改进和创新。仅举几个例子：

如用户体验上，用户不用关注去哪里找影片，系统会自动搜索互联网上评论最佳、清晰度最高、播放最快的片源推荐给客户；人机交互可通过手机、PAD平板电脑操控界面，实现多屏同步；

公司有现成的产品，如果你感兴趣，可以和我们约个时间过来参观试用一下，我们相信不会让你感到失望。

我们的一款产品已经上市了，有一个很好听的品牌名字叫 Dotee 豆米。

我们通过互联网销售产品，云深网络官方网站 www.atyun.net 可以下订单购买产品，在线支付，快递送货。

计划下个月网站正式上线运营。

##我们需要什么样的 RoR 开发工程师
* 有志于成为一名优秀的 RoR 工程师
* 要有至少 6 个月 RoR 开发经验，并对新技术、新领域感兴趣

* 要有一定的英文阅读能力

##工作职责
使用 RoR 开发基于 Web 的互联网网站，包括但不限于：
* 销售产品的在线购物网站；
* 为互联网和智能电视的 App 应用搭建应用商店，并提供丰富多彩的互联网内容服务；
* 类似 Path 的朋友小圈子分享的 SNS 社区，在朋友之间分享喜欢的视频信息；
* 提供视频领域的垂直搜索服务，让用户能够搜索到自己感兴趣的互联网视频；

如果感兴趣，你可以参与 Python 开发互联网和智能电视的 App 应用，或 Android 和 iOS 开发手机或者平板电脑 App 应用，甚至可以使用 C 或 C++ 参与操作系统定制和 SDK 开发和维护。

我们认为没有一种语言或技术能够解决所有的问题，所以鼓励更多开发人员能够成为技术多面手。

我们允许试错，鼓励尝试新技术和新思维，更欢迎将新东西引入日常工作。

便利性：比电脑外接屏幕上看电视更方便，更快捷

1 与电视机的物理特性有关，观看距离长，
比如沙发到电视机屏幕的距离，一般有2米左右吧。
大部分互联网网站（包括视频网站），应用程序的界面或布局主要为电脑／手机／平板电脑，近距离观看开发的。如果您坐在沙发上，即使您配上一个无线鼠标，看到电视即屏幕上密密麻麻的文字和超链接，估计您要经常蹲到电视机屏幕跟前，才方便继续下一步操作。

2 对不熟悉电脑的用户来说，直接操作电脑有困难。
我们这一代年轻人大都熟悉使用电脑访问互联网。其实身边有很多人，比如父母或者上了年纪的人，他们其实也想访问或者接触互联网，比如看互联网的丰富的电视剧，看天气或者新闻咨询，看看异地子女放在网络上的照片。。。但是对于他们来说，操作电脑的确复杂些。
我们的盒子就像手机／平板电脑，里面可以运行专门的 App 应用。他们可以做的沙发上，通过简单摇控器点点操作，就可以在电视机上完成上述需求。就和平常看电视那样简单。

的确有很多 IT 技术高手，乐此不疲地 DIY 电脑外接屏幕上看电视。但是如果有一个盒子，能够很方便快捷地在电视大屏幕上看网络视频，访问互联网的内容。我想大部分人更乐意接受后者。

另外举个教育方面的例子，互联网有很多好的教育资源，但作为父母更愿意让孩子通过 平板电脑 App 应用访问互联网，而不放心孩子用电脑上网，因为孩子更很容易通过电脑访问其它不良的信息。
同样，在电视机开发教育类的 App 应用，容易让父母接受，也是同样的道理。

产品能否有用，要看能否满足目标用户群体的需求。

好像有点像，有人既然有电脑了，为何会另外要买 iPad 平板电脑。。。可能有点牵强，如不正确，请见谅。
